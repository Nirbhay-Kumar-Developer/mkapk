This tool mkapk is designed to be resource efficient.
Using it you can create Android Apps directly on your Android phone using termux.

How to use: General instructions

Step-1: Install a text editor for writing your code.
        Use any text editor you are comfortable with or use nano in termux instead.

Note: Use a text editor which has a built-in file explorer to make the user experience better.
      And make your projects in the internal storage (/storage/emulated/0) so they are accessible through your text editor.

Step-2: Install Termux from F-droid (<weblink>). 

Step-3: Run this command

#######################################
Note: If you are a new termux user, run 
         bash mkapk-install.sh
#######################################

        <your shell name> mkapk-install.sh
        
        For example:
        (i) for sh users
         sh mkapk-install.sh

       (ii) for bash users
         bash mkapk-install.sh

      (iii) for zsh users
         zsh mkapk-install.sh

Now, congratulations, you have now installed a environment where you can build Android apps directly on your Android phone.

######################################
Uasge Instructions:

       To make a new project:
        mkproj

       To cd to your project directory:
        cdproj

       To build the debug apk:
        mkapk -d

       To build the release apk:
        mkapk
########################################

########################################################################
Project Directory Structure:
    src ---> your java code here
    res ---> resources directory
    bin ---> your compiled files and final apk here
    Releases ---> your released apk files here
    project$rootfs$ ---> all other directories and Android Manifest here
########################################################################

Special Note: You can customize anything you want, any scripts, if you are new and something gets wrong in customization don't be scared, just rerun the installation script, thus will not delete your project data. But always keep a backup of your project files. Keep a backup of your  installation script and unmodified mkapk scripts in the same directory.
